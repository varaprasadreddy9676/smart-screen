# Hackathon Demo Script

This is the shortest high-signal demo flow for judges.

Total target time: `2-3 minutes`

## Demo Goal

Show that the product does more than record a screen. It understands the demo and helps turn it into something presentable.

## Suggested Script

### 1. Start With The Problem

Say:

> Most screen recorders only capture. People still need to trim dead time, add captions, explain what happened, and make the video watchable.

Then show a raw recording in the editor.

### 2. Show Original Preview

Use the top-bar `Compare Original` control and keep the preview in `Original Preview`.

Call out:

- plain raw recording
- no zooms
- no captions
- no polish

### 3. Add Narration Understanding

If you have a recording with voice:

- click `Transcribe Audio`
- or load a prepared transcript

Open the transcript review briefly and show:

- editable transcript
- captions can be previewed or burned in

Say:

> The transcript is not just for subtitles. It becomes another signal for understanding the demo.

### 4. Run One-Click Polish

Click `One-Click Polish Demo`.

Then switch to `Polished Preview`.

Call out:

- calmer zooms
- click emphasis
- captions
- better pacing

This is the main “aha” moment.

### 5. Show AI Assist

Click the persistent `AI Assist` button in the top bar.

Show:

- BYOK model choice
- OpenAI or Ollama
- optional local-first workflow

Then run AI refine on the current recording.

Call out:

- summary
- refined steps
- selective apply for zooms or trims

Say:

> AI here is optional and reviewable. The app still works offline, but AI can improve structure and narration understanding.

### 6. Show Why This Is Better

Use the compare toggle again:

- `Original Preview`
- `Polished Preview`

This makes the transformation obvious in seconds.

### 7. Export

Show:

- MP4 export
- SRT / VTT subtitle export
- burned-in captions

If keystrokes are present, point out keystroke overlays too.

## Best Recording For The Demo

Use a short recording that includes:

- at least 2 or 3 meaningful clicks
- spoken instructions like “click export”, “look at this chart”, or “open settings”
- one short dead pause
- one keyboard shortcut if possible

That lets you demo:

- native click telemetry
- transcript grounding
- zoom polish
- captions
- AI refinement

## Judge-Friendly Talking Points

- `Local-first`: useful even without any API key
- `Hybrid AI`: works with OpenAI or Ollama
- `Speech-grounded`: understands narration, not just cursor motion
- `Reviewable`: AI suggestions are not blindly applied
- `Demo-ready`: captions, click emphasis, keystrokes, trims, export

## If You Only Have 60 Seconds

1. Show `Original Preview`
2. Click `One-Click Polish Demo`
3. Show `Polished Preview`
4. Open `AI Assist`
5. Say: “It works offline, and AI is optional for deeper refinement.”
